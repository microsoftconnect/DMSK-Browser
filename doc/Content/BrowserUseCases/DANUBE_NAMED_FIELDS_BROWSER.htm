Naming fields for voice navigation

You can enable specific controls for named field navigation, so that the user can navigate to a specific field by voice. Add the attribute data-nusa-concept-name to the control and set its value to the field name that the user should say; for example, data-nusa-concept-name="medical history". This attribute will be part of the go to command to navigate to that field; the user will be able to say go to medical history.

Use distinct and intuitive concept names; avoid duplicate or similar sounding concept names.

Do not use:

Abbreviations or book title capitalization

Punctuation marks (for example, ", !, ?, etc.)

Symbols (for example, +, -, &, $, etc. - use words such as "and" instead of symbols such as "&")

Digits (use 'twelve' and not '12')

Full phrases such as "move cursor to medical history".

Before you release your integration, test that navigating to named fields using the go to voice command works as intended.
