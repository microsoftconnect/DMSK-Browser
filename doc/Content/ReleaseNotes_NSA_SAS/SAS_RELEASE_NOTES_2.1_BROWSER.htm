Release Notes for 360 | SpeechAnywhere Services 2.1 (Browser edition)

New in version 2.1

  • Microsoft Silverlight is no longer used for the latest versions of Mozilla Firefox and Google Chrome browsers. Safari and Internet Explorer continue to require Microsoft Silverlight for speech recognition.
  • The SAS browser SDK now also supports Single page applications. For more information on how to enable speech recognition for such applications, see the 'Single page applications' topic in the SDK help.

  • Network connection reliability has been improved.

  • In Mozilla Firefox and Google Chrome browsers the NUSA_startRecording() function now works without the user first having to start recording manually.

Important information in version 2.1

  • 360 | SpeechAnywhere Services 2.1 Browser edition is compatible with server versions 2.0.

Important: You must ensure that you have upgraded your SAS server to a compatible version in order to use the SAS Browser SDK.

  • 360 | SpeechAnywhere Services 2.0 Server is compatible with client versions 1.5, 1.6, 2.0 and 2.1.
  • Do not change or hide SAS Browser SDK GUI elements. Changing or hiding these elements risks the stability and compatibility of the SDK and Nuance cannot provide support for such integrations.
  • Web browser debugging consoles or debugging proxies (such as, Fiddler) cannot be used with speech recognition. The performance impact incurred by such infrastructure leads to unreliable speech recognition.
  • If a web page is not hosted on HTTPS, Google Chrome will ask for microphone access every time the page is loaded or changed.

Known issues

  • In Internet Explorer 11 the cursor position is incorrect after copied text has been pasted in. [DNB-1434]
  • On OS X, speech recognition can be temporarily unavailable after switching between browsers. Solution: Refresh the page. [PSP00070425]
  • Internet Explorer: SAS Browser edition does not work correctly with Compatibility View turned on. Solution: Turn off Compatibility View. [DNB-1457, DNB-1458]
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [DNB-328]
  • The Record button does not work on MacBook via Safari browser. Workaround: For the Silverlight plugin, activate Run in Unsafe Mode in the Manage Website Settings (via the Preferences menu, Security, tab of the Safari browser). [DNB-1423]

  • This release does not support mobile devices; including browsers (Firefox and Chrome) that do not use Silverlight.

  • iOS mobile devices do not work using any browser due to Apple limitations on third-party browsers (https://developer.chrome.com/multidevice/ios/overview) and lack of the corresponding API support, in Safari, by Apple (http://caniuse.com/#feat=stream%26feat_search=getUserMedia).

Infrastructure

Infrastructure no longer supported

  • 360 | SpeechAnywhere Services (Browser edition) 2.1 client applications do not work with any server version prior to 2.0.
  • Mozilla Firefox 24.x and lower are no longer supported.
  • Google Chrome 29.x and lower are no longer supported.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 2.2 all version 1.5 client-based applications will stop working.
  • Mozilla Firefox 36.x and lower will no longer be supported.
  • Google Chrome 40.x and lower will no longer be supported.

Important: Microsoft are changing their Internet Explorer support policies from January 12 2016 (see: https://support.microsoft.com/en-us/gp/microsoft-internet-explorer). From this date only Internet Explorer 10 and Internet Explorer 11 will be supported by Microsoft and therefore by SAS. From this date Internet Explorer 8 and Internet Explorer 9 will no longer be supported by SAS.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s end-of-life and Nuance's policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
