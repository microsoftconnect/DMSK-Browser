Release Notes for 360 | SpeechAnywhere Services 1.6 Service Release 19 (Browser edition)

Solved issues in service release 19

In Microsoft Internet Explorer 9, the cursor no longer jumps to the end of the field when recording is started. [DNB-2655, DNB-2598]

In Microsoft Internet Explorer 11, the cursor is no longer in an incorrect position after typing <Enter> between two dictations. [DNB-1867]

Manually changing the focus to another speech-enabled field no longer results in recognition inaccuracies in the next utterance. [DNB-2575]

Network connection reliability has been improved. [DNB-1476]

The user can now select text via voice in a text control that has the voice focus, but not the GUI focus. [DNB-1859]

The undo that voice command now works properly in single line fields. [DNB-1449]

Note: For information on previous service releases, see below.

Important information in version 1.6

  • In order to take advantage of these improvements, 360 | SpeechAnywhere Services 1.6 SR 10 Server is highly recommended with this version of the Browser edition.
  • 360 | SpeechAnywhere Services 1.6 Server is compatible with client versions 1.4, 1.5 and 1.6.
  • 360 | SpeechAnywhere Services 1.6 Browser edition is compatible with server versions 1.4, 1.5 and 1.6.
  • Do not change or hide GUI elements. Changing or hiding GUI elements risks the stability and compatibility of the SDK and Nuance cannot provide support for such integrations.

New in version 1.6

  • Voice editing functionality has been extended with select & say voice commands (Dragon-based systems only).
  • Text formatting has been improved. When text is inserted, the spacing is correctly updated and surrounding text is correctly capitalized (Dragon-based systems only).
  • SAS clients can now use Dragon customization packs (.dat files) containing Text-and-Graphics/auto-texts without lists. You can import and share .dat files via the Nuance Management Console.
  • SAS Browser edition has a new method to start recording: NUSA_startRecording().

Known issues

  • In Internet Explorer 11 the cursor position is incorrect after copied text has been pasted in. [DNB-1434]
  • On OS X, speech recognition can be temporarily unavailable after switching between browsers. Solution: Refresh the page. [PSP00070425]
  • Internet Explorer: SAS Browser edition does not work correctly with Compatibility View turned on. Solution: Turn off Compatibility View. [DNB-1457, DNB-1458]
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [DNB-328]
  • The Record button does not work on MacBook via Safari browser. Workaround: For the Silverlight plugin, activate Run in Unsafe Mode in the Manage Website Settings (via the Preferences menu, Security, tab of the Safari browser). [DNB-1423]

Infrastructure

Infrastructure no longer supported

  • Version 1.2 client applications will stop working with version 1.6 servers.
  • Mozilla Firefox 17.x and lower are no longer supported.
  • Google Chrome 24.x and lower are no longer supported.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 1.7 all version 1.3 client-based applications will stop working.
  • Mozilla Firefox 24.x and lower will no longer be supported.
  • Google Chrome 29.x and lower will no longer be supported.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s end-of-life and Nuance's policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.

Service release history

Solved issues in service release 6

  • Cursor position issues when selected text is overwritten with recognition results have been fixed. [PSP00076214, PSP00076755, PSP00076918, PSP00077222]
  • The handling of line breaks and word wrapping in multiline fields has been made more consistent. [PSP00076438, PSP00076456, PSP00076699, PSP00076869, PSP00076916, PSP00077146, PSP00077243, PSP00077352, PSP00077275]
  • Text formatting and line break handling have been improved in fields that contain a combination of typed/pasted and recognized text and when recognition results are inserted into existing text. [PSP00076759, PSP00076761, PSP00076768, PSP00076774, PSP00077212, PSP00077275, PSP00077285, PSP00077306, PSP00077501]
  • The handling of field focus changes while recognition results are pending in Google Chrome has been improved. [PSP00075963]
  • When the current word is selected via the corresponding voice command (for example, select this word) while recognition results are pending, the recognized text will now be inserted with the last word selected. [PSP00075987]
  • Voice commands that select or format all of the text in a field now work correctly in Firefox. [PSP00076767]
  • Connections between the Browser edition and the SAS Server are now more robust; connections in a network environment that is not ideal now perform better. SAS Sever version 1.6 SR 10 is required. [PSP00077652 PSP00078628 PSP00078631 PSP00078683 PSP00078558]
