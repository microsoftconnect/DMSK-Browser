Release Notes for Dragon Medical SpeechKit 3.0 (Browser edition)

Dragon Medical SpeechKit is a component of the Nuance Healthcare Development Platform. Rebranding the components is currently an ongoing activity. Dragon Medical SpeechKit was previously called 360 | SpeechAnywhere Services; please be aware that references to 360 | SpeechAnywhere Services 3.0 apply to this release.

Important information

  • Dragon Medical SpeechKit 3.0 (Browser edition) requires Dragon Medical Server 3.0. Make sure to upgrade the server before you upgrade to this Browser edition.
  • Dragon Medical Server 3.0 is compatible with Browser editions 2.0, 2.1 and 3.0.
  • Information on the voice commands changed with each version of Dragon Medical is now available in the end user help.
  • On the speech bar the colors for recording on/off have changed from red (on) - gray (off) to green (on) - red (off).
  • Do not change or hide Browser edition GUI elements. Changing or hiding these elements risks the stability and compatibility of the product; Nuance cannot provide support for such integrations.
  • Web browser debugging consoles or debugging proxies (such as, Fiddler) cannot be used with speech recognition. The performance impact incurred by such infrastructure leads to unreliable speech recognition.

New in version 3.0

Manage Vocabulary (server version 3.0 required)

Dragon-based systems
  • Users can now search for words on the Manage Vocabulary page.
  • Users can now filter words on the Manage Vocabulary page based on the level (user/group/site) for which they were added.
  • New Modify Word screen: Users can now edit the spelling and pronunciation of words in their personal lexicon.
  • Users can now record how a word is pronounced.
  • New voice command to record how a word is pronounced: train word.

Personal auto-texts (server version 3.0 required)

All systems
  • Users can now create auto-texts via the personalization and help window, Manage Auto-texts page. This new functionality is available to desktop users and mobile device users with a Dragon Medical One license.
  • New voice command: make that an auto-text.

Speech recognition (server version 3.0 required)

Dragon-based systems
  • New English, Dutch, Flemish and German voice commands to form compound words. English examples: compound, compound that, compound selection.
  • New German voice commands to separate compound words: trenne das, Auswahl trennen, trenne Auswahl.
  • Improved correction alternatives for Dutch, French and German in the correction menu.
  • SpeechAnywhere now intelligently identifies sentence separators; this means voice commands for selecting sentences and navigating to sentences work more reliably.
All systems
  • New voice commands to navigate text: go to beginning of sentence, go to end of sentence, go to beginning of paragraph, go to end of paragraph.

Browser edition

  • The Browser edition can now speech-enable a specific set of edit controls on a page; pass an array of edit control elements or their container to NUSA_initialize().
  • The Browser edition can now query if the user's browser is supported; use NUSA_isBrowserSupported().

On-premises systems (server version 3.0 required)

  • Dragon Medical systems now remove speech recognition data from the Dragon Medical File Share when the data is no longer needed for improving the recognition quality.

Known issues

  • In Internet Explorer 11 the cursor position is incorrect after copied text has been pasted in. [DNB-1434]
  • On OS X, speech recognition can be temporarily unavailable after switching between browsers. Solution: Refresh the page. [PSP00070425]
  • Internet Explorer: The Browser edition does not work correctly with Compatibility View turned on. Solution: Turn off Compatibility View. [DNB-1457, DNB-1458]
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [DNB-328]
  • The Record button does not work on MacBook via Safari browser. Workaround: For the Silverlight plugin, activate Run in Unsafe Mode in the Manage Website Settings (via the Preferences menu, Security, tab of the Safari browser). [DNB-1423]

  • This release does not support mobile devices; including browsers (Firefox and Chrome) that do not use Silverlight.

  • iOS mobile devices do not work using any browser due to Apple limitations on third-party browsers (https://developer.chrome.com/multidevice/ios/overview) and lack of the corresponding API support, in Safari, by Apple (http://caniuse.com/#feat=stream%26feat_search=getUserMedia).

  • In Google Chrome and Mozilla Firefox the page must be refreshed if the microphone is unplugged and plugged in again. [DNB-4123]

Infrastructure

Infrastructure no longer supported

  • Microsoft Internet Explorer 10 and lower (see Microsoft Lifecycle Support policy for Internet Explorer: https://support.microsoft.com/en-us/gp/microsoft-internet-explorer)
  • Dragon Medical SpeechKit (Browser edition) 3.0 applications do not work with any server version prior to 3.0.
  • Mozilla Firefox 44 and lower are no longer supported (see Mozilla Firefox ESR Overview: https://www.mozilla.org/en-US/firefox/organizations/faq/).
  • Google Chrome 40.x and lower are no longer supported.
  • In Dragon Medical SpeechKit 3.0 all version 1.4 client-based applications stop working.

Infrastructure changes planned for the next release

  • In Dragon Medical Server 3.1 all version 1.5 and 1.6 client-based applications will stop working.
  • Google Chrome 47 and lower will no longer be supported.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s end-of-life and Nuance's policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
