Release Notes - 360 | SpeechAnywhere Services 1.3 Service Release 11

This is a service release for 360 | SpeechAnywhere Services Browser edition.

New features

Your application can now be notified about events related to individual text controls (within a VuiForm):

function NUSA_onProcessingStartedForElement(element)

function NUSA_onProcessingFinishedForElement(element)

Solved issues

Application commands are now recognized after forms are reinitialized.

The documentation on how to speech-enable specific controls has been improved.

The Events use case documentation now contains information about text control events.

Feedback bubble positioning has been improved.

Concept name field attributes containing non-alphanumeric characters are now handled properly.

The reliability of the start recording procedure has been improved.

Stopping recording and leaving the application open without using it for several minutes no longer results in an error.

Error handling was improved for the functions NUSAI_addLogoToElement and NUSAI_EventUtteranceProcessed.

Starting recording when the cursor is not in a speech-enabled field now inserts the recognition result in the first speech-enabled field.

VUI forms with more than 25 fields are now properly handled.

Important: Hidden or disabled fields are no longer speech-enabled.

Known issues

In Internet Explorer 7 the feedback bubble is sometimes not displayed properly. We recommend using Internet Explorer 8 or higher.

Occasionally the connection to the USB recording device is lost and cannot be re-established. Workaround: plug the device into another USB port.

In hosted systems the first 2 utterances of a first time user are not reliably recognized.

In on-premises systems the SpokenText parameter of CommandRecognized events is always empty.
