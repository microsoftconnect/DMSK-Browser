Release Notes for 360 | SpeechAnywhere Services 1.5 Service Release 9 (Browser edition)

Solved issues

Recognized text is now correctly inserted at the cursor position in CKEditor applications. [PSP00074209]

Error messages are now displayed in Portuguese. [PSP00071035]

How to stop recording explicitly from code is now documented in the SDK help. [PSP00071583]

Important information

  • In Nuance-hosted systems, when your application connects to the 360 | SpeechAnywhere Services 1.5 Server your application will automatically be directed to the 1.5 client version. In on-premises systems, the 360 | SpeechAnywhere Services 1.5 Server is compatible with client versions 1.3 and 1.4.
  • In NMC "customer token" has changed to "organization token".

Known issues

  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]

Infrastructure no longer supported

  • Version 1.1 client-based applications have stopped working.
  • Mozilla Firefox 17.x and lower
  • Google Chrome 24.x and lower

Infrastructure changes planned for the next release

  • Support for Mozilla Firefox 17.x and lower will be removed.
  • Support for Google Chrome 24.x and lower will be removed.
  • In 360 | SpeechAnywhere Services 1.6 all version 1.2 client-based applications will stop working

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s end-of-life and Nuance's policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
